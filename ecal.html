Sistem Rekomendasi Musik

Sebuah Rencana Proyek Interaktif untuk Skripsi "Sistem Rekomendasi Untuk Pemilihan Musik Menggunakan Metode Knowledge-Based Recommendation"

Metodologi Inti: KBRS Hibrida

Proyek ini mengusulkan model Knowledge-Based Recommendation System (KBRS) hibrida yang canggih untuk mengatasi skenario pengguna yang berbeda. Pendekatan ini secara cerdas menangani masalah 'cold-start' untuk pengguna baru sambil memberikan personalisasi mendalam untuk pengguna terdaftar.

Untuk Tamu: Constraint-Based

Sistem merekomendasikan lagu dengan menerjemahkan input sementara pengguna (mood, aktivitas) menjadi batasan eksplisit. Ini cepat, transparan, dan tidak memerlukan riwayat pengguna.

Contoh Alur Logika:
1. Input: Pengguna memilih Mood 'Senang' & Aktivitas 'Pesta'.
2. Terjemahan: Sistem membuat batasan: `mood IN ['happy', 'energetic']` DAN `tempo > 120`.
3. Eksekusi: Query SQL dijalankan untuk mencari lagu yang cocok.
SELECT * FROM songs WHERE mood_tag IN ('happy', 'energetic');
4. Hasil: Daftar lagu yang relevan ditampilkan.

Untuk User: Decision Tree

Untuk pengguna terdaftar, sistem 'mempelajari' preferensi mereka dengan melatih model Decision Tree. Model ini mengubah riwayat rating menjadi serangkaian aturan IF-THEN yang dapat diinterpretasikan.

Contoh Aturan Terpelajar:
1. Fitur: `user_genre`, `user_mood`, `song_genre`, `song_mood`.
2. Target: `rating > 3` (Suka) / `rating <= 3` (Tidak Suka).
3. Aturan: Pohon keputusan menghasilkan aturan seperti:
IF user_prefers('rock') AND song_mood('energetic') THEN predict('Suka')
4. Penjelasan: Memungkinkan fitur "Mengapa lagu ini direkomendasikan?".

Rencana Evaluasi Kinerja

Evaluasi sistem akan dilakukan secara komprehensif, menggabungkan metrik kuantitatif untuk akurasi, metrik kualitatif untuk kualitas pengalaman, dan survei kepuasan pengguna untuk umpan balik langsung.

Metrik Kuantitatif & Kualitatif

Precision@K & Recall@K: Mengukur akurasi dasar dari K rekomendasi teratas.
NDCG: Metrik peringkat canggih yang menilai posisi item relevan.
Diversity: Mengukur seberapa bervariasi lagu dalam daftar rekomendasi.
Novelty & Serendipity: Menilai kemampuan sistem merekomendasikan lagu baru yang mengejutkan namun relevan.

Proses "Plug-and-Play"

  1. Prasyarat: Dosen penguji hanya perlu menginstal Docker Desktop.
  2. Simpan Image: Semua base image (Python, Postgres, dll.) disimpan sebagai file `.tar` di dalam CD untuk operasi offline.
  3. Jalankan Skrip: Penguji cukup mengklik `start-project.bat` (Windows) atau `start-project.sh` (Mac/Linux).
  4. Otomatisasi: Skrip secara otomatis akan:
    • Memuat image Docker dari file `.tar`.
    • Menjalankan `docker-compose up` untuk membangun dan memulai semua layanan (DB, Backend, Frontend).
    • Membuka browser ke `http://localhost:3000`.
